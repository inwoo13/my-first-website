HTML로 돌아가기

웹페이지를 만드는 코드는 “html”

html은 쉽고, 중요함
웹페이지는 여기저기 깔려있고, 쉬워서 html이 중요하고 선행되야함.
웹페이지를 만들기 위해서는 html을 배워야 한다.

html을 공부하기 위해서 웹브라우저가 있어야 하고,
html을 컴퓨터도 이해하고 사람도 이해할 수 있는 컴퓨터 언어의 문법에 맞게 코드를 작성해야 하는데,
코드를 작성하는 프로그램이 필요하다.
그러한 프로그램은 에디터라고 하고 모든 컴퓨터에는 에디터가 있음.
에디터 : 메모장, 텍스트 편집기, gedit, nano, vim

각각의 에디터들은 코드를 작성하는 용도가 아니므로 코드를 작성하는데 최적화된 전문적인 도구를 사용할 것임.(아톰 같은거)
에디터마다 사용법은 다르며 아톰(Atom)은 깃허브에서 만든거임.

자신에게 맞는 에디터를 찾으려면
best editor (최신년도)
로 구글에 검색해서 찾으면 됨.

현 시점에서는 아톰을 사용할 예정

Atom 사용
폴더를 생성 -> 아톰에서 open folder를 클릭
왼쪽 오른쪽으로 창이 나눠지는데 왼쪽은 폴더(디렉토리)를 관리하는 화면,
웹페이지는 확장자가 html임

웹브라우저와 같은 컴퓨터에 있는 파일인 1.html을 읽으려면 파일열기라는 기능을 사용하면 됨.
윈도우에서는 ctrl+O를 누르면 파일열기.
(이 기능같은 경우에는 Edge에서는 사용이 잘 안됨. Chrome이나 internet explore사용을 하는게 좋음.)
내가 생성한 파일을 웹브라우저에 띄우고 싶을 때 웹브라우저에서 파일열기를 통해서 파일을 선택하면 웹브라우저에 띄워진다.

위에서 아톰을 이용하여 한 것들은 메모장도 에디터이므로 똑같이 활용할 수 있음.

에디터에 작성한 것이 원인, 웹브라우저에 띄워지는 것이 결과이다.
에디터에 작성한 것이 코드, 소스(html이라는 언어에 문법에 맞게 작성된 코드, 소스임)

TAG : 설명의 역할(굵게하고 밑줄긋는 등의 역할로)
앞에있는 태그는 열리는 태그 뒤에 있는 태그는 닫히는 태그이다.
닫히는 태그는 /를 넣는다. 태그는 중첩해서 사용이 가능하다.

기본문법 태그
<.strong> <./strong> : strong사이에 있는 문자를 굵게 표시
ex. 나의 이름은 <.strong>임용수<./strong>입니다.
<.u> <./u> : u사이에 있는 문자를 밑줄표시
ex. 나의 이름은 <.u>임용수<./u>입니다.

한글이 깨지는 경우
<.meta charset=“utf-8”>를 추가. 파일을 저장하면 UTF-8로 저장이 됨. 그럼 웹 브라우저에게 UTF-8로 페이지를 열라고 해줘야 한다. 그걸 브라우저에게 알려주는 코드가 <.meta charset=“utf-8”>이다.


웹 브라우저의 페이지 소스를 보는 방법
마우스 오른쪽 클릭 -> 페이지 소스 보기(페이지 원본) 클릭
소스를 보다보면 모르는 태그를 접하게 되는데, 그것을 알고싶으면 html h1 tag라는 식으로 검색을 해주면 된다.
html에는 150개 이상의 태그가 존재함. 자신에게 필요하다고 생각되는 태그들만 알고있어도 충분히 html을 잘 활용할 수 있음.

줄바꿈 태그
<.p> <./p>:엔터 두 번의 용도. 어디부터 어디까지가 한 단락인지 알 수 있으므로 열리는 태그와 닫히는 태그가 존재. 웹페이지를 정보로서 가치있게 만드는 태그이다. 단락이다라는 것을 의미론적으로 표현해줄 수 있음.
<.br>:엔터 한번의 용도. 줄바꿈이라는 시각적인 의미만 있으므로 닫히는 태그를 쓰지 않음.

태그 이름만으로는 정보가 부족하여 속성(attribute)를 만듦.
unsplash.com => 저작권에 구애받지않고 사용할 수 있는 이미지들이 있다.
<.img src=“” width=“”> 따옴표 안에 원하는 이미지의 주소를 적으면 웹브라우저는 주소를 이미지로서 표현
scr는 source의 줄임표현. src가 속성임. 속성의 위치는 상관없음.

<.parent>
<.child><./child>
<./parent>
태그가 서로가 포함관계로 연결되어 있을 때, 포함하고 있는 태그를 부모태그, 포함된 태그를 자식태그라고 한다.

들여쓰기 : tab키
여러줄에 같은 내용 쓰기 : ctrl누른 상태로 내용 넣을 곳에 선택

<.li> <./li>:리스트 생성. li와 같은 항목들은 어디부터 어디까지가 서로 연관된 항목인지를 경계를 짓기위한 부모태그인 <.ul> <./ul>이 있어야 함. 반드시 부모태그를 가지고 있어야 함.
<.ul> <./ul> unordered list
<.ol> <./ol> ordered list, 항목의 앞에 자동으로 숫자가 생성

<.body> <./body> 본문을 묶음. 본문의 시작과 끝에 꼭 넣어줌.
<.head> <./head> 본문(바디)를 설명하는 태그를 묶어줌. 꼭 한다.
이 모든걸 묶는게 <.html> <./html>

관용적으로 제일 먼저 제일 위에 쓰는거
html specification -> 검색하면 html 사용설명서가 나옴
<.a> <./a>: 링크를 의미.

표 만들기
<.td><./td> : 각 항목을 표의 항목으로 만들기
<.tr><./tr> : 같은 행으로 묶음
<.table><./table> : 표를 만들음
<.table border="2"> : html만 사용할때 표의 테두리를 만들어줌.
